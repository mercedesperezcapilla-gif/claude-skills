# fs-governance-tier-checker

Tiers AI use cases by governance significance for financial services professionals. Uses output function — not user audience — to classify use cases as Productivity, Regulated Input, or Externally Consequential. Returns routing logic, governance considerations, and preparation guidance for your compliance conversation.

---

## How to install

Copy this folder into your Claude Code skills directory:

```
skills/fs-governance-tier-checker/  →  ~/.claude/skills/fs-governance-tier-checker/
```

Then use it by typing `/fs-governance-tier-checker` in Claude Code.

---

## How to use

Describe your AI use case and the skill will return:

1. **Suggested tier** — Productivity, Regulated Input, or Externally Consequential
2. **Routing logic** — why it lands at that tier
3. **Typical governance considerations** — what your firm will likely want to see
4. **Before your governance conversation** — what to prepare and who to involve

---

## The three tiers

| Tier | Label | When it applies |
|---|---|---|
| 1 | Productivity | Output does not affect regulated decisions, customer outcomes, or counterparty exposure |
| 2 | Regulated Input | Output influences operational or regulated workflows but human review remains |
| 3 | Externally Consequential | Output directly drives outcomes with external or regulatory significance |
